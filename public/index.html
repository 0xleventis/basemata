how it works

      

        

          
step_01

          
Authenticate

          
Authorize with public_repo scope. Your identity attaches to every PR — anonymous PRs are often ignored or rejected.

        

        

          
step_02

          
Configure token

          
Upload logo, paste contract address (auto-validated on Base RPC), add socials and tags. One form for all registries.

        

        

          
step_03

          
Select targets

          
Choose which DEX token lists to target. Each maintains its own GitHub repo as the source of truth for its frontend.

        

        

          
step_04

          
Auto-propagation

          
Agent forks, commits, and opens PRs. On merge, the DEX reads from that repo and your token appears automatically.
